# 爬取数据解析文档

本文档说明系统从 **哔哩哔哩（B站）** 采集的视频数据的来源接口、字段含义、解析与清洗规则，
以及最终落地的 CSV 文件结构。所有采集结果统一保存在 `backend/data/videos.csv`。

---

## 一、数据来源接口

采集流程依次调用以下 3 个 B 站 Web 端公开接口：

| 步骤 | 接口 | 用途 |
| --- | --- | --- |
| 1 | `https://www.bilibili.com` | 首次访问获取匿名 Cookie（buvid 等），降低风控概率 |
| 2 | `https://api.bilibili.com/x/web-interface/nav` | 获取 WBI 签名所需的 `img_key` / `sub_key` |
| 3 | `https://api.bilibili.com/x/web-interface/wbi/search/type` | 按关键词分页搜索视频列表 |
| 4 | `https://api.bilibili.com/x/web-interface/view` | 按 BV 号查询单个视频完整统计（点赞/投币/转发等） |

### 关于 WBI 签名
B 站搜索接口要求对请求参数做 **WBI 签名**（生成 `w_rid`），否则返回 `code=-412`（风控）。
签名算法（见 `backend/crawler.py`）：
1. 从 nav 接口取 `img_key`、`sub_key`，拼接后按固定的 64 位混淆表重排，取前 32 位得 `mixin_key`；
2. 请求参数加入时间戳 `wts`，按 key 升序排列并 URL 编码；
3. `w_rid = md5(排序后的query字符串 + mixin_key)`。

## 二、CSV 字段含义

文件：`backend/data/videos.csv`（编码 `utf-8-sig`，可被 Excel 直接打开）

| 字段名 | 中文含义 | 来源 | 类型 | 说明 |
| --- | --- | --- | --- | --- |
| `bvid` | 视频 BV 号 | 搜索接口 `bvid` | 字符串 | 视频唯一标识，去重主键之一 |
| `title` | 视频标题 | 搜索接口 `title` | 字符串 | 已去除 `<em>` 高亮标签与多余空白 |
| `url` | 视频链接 | 由 bvid 拼接 | 字符串 | `https://www.bilibili.com/video/{bvid}` |
| `author` | UP主（博主）昵称 | 搜索接口 `author` | 字符串 | |
| `mid` | UP主用户ID | 搜索接口 `mid` | 字符串 | UP主在 B 站的唯一 ID |
| `play` | 播放量 | view 接口 `stat.view`（兜底用搜索 `play`） | 整数 | 已做"万/亿"换算 |
| `danmaku` | 弹幕数 | view `stat.danmaku`（兜底搜索 `video_review`） | 整数 | |
| `reply` | 评论数 | view `stat.reply`（兜底搜索 `review`） | 整数 | |
| `favorite` | 收藏数 | view `stat.favorite`（兜底搜索 `favorites`） | 整数 | |
| `coin` | 投币数 | view `stat.coin` | 整数 | 需开启"补全互动数据" |
| `share` | 转发(分享)数 | view `stat.share` | 整数 | 需开启"补全互动数据" |
| `like` | 点赞数 | view `stat.like` | 整数 | 需开启"补全互动数据" |
| `duration` | 时长(秒) | 搜索接口 `duration` | 整数 | 由 "MM:SS" 转为秒 |
| `pubdate` | 发布时间 | 搜索接口 `pubdate` | 字符串 | unix 时间戳转 `yyyy-mm-dd HH:MM:SS` |
| `keyword` | 采集关键词 | 采集参数 | 字符串 | 本条数据是用哪个关键词搜到的 |
| `crawl_date` | 采集日期 | 系统生成 | 字符串 | `yyyy-mm-dd`，用于按日期查看/去重 |

> **说明**：搜索接口本身不返回 `点赞/投币/转发`，需勾选"补全互动数据"后由 view 接口逐条补全。
> 若关闭补全，这三列为 0，采集速度更快。

## 三、数据清洗规则（实训模块二）

清洗逻辑见 `backend/cleaner.py`：

1. **数量单位归一化**：把带单位的字符串统一转为整数。
   - `"1.1万"` → `11000`
   - `"2.3亿"` → `230000000`
   - `"1,234"` → `1234`
   - `"-"` / `""` / `None` → `0`
2. **标题净化**：去除搜索结果中的 `<em class="keyword">...</em>` 高亮标签与连续空白。
3. **类型保证**：读取 CSV 时强制把 8 个数值列转为 `int64`，非法值填 0。

---

## 四、去重与持续更新规则

- 去重主键：`bvid + crawl_date`。
- 同一天对同一视频多次采集 → 保留**最新**一次（统计值随之更新），实现"持续更新数据"。
- 不同 `crawl_date` 的同一视频视为不同记录，便于观察播放量随时间的变化、按日期回看。

---

## 五、采集方式

1. **手动采集**：管理员在「爬虫基本设置」页配置关键词/页数/间隔后点击"开始采集"，后台线程执行，前端每 2 秒轮询进度。
2. **自动采集**：APScheduler 每日凌晨 02:00 自动采集前一天数据（`crawl_date` 标记为前一天），见 `backend/scheduler.py`。
