============================================================
  B站数据分析与AI知识库系统 —— 启动说明
============================================================

【说明】
本项目 = 后端(FastAPI) + 前端(React)，需要各开一个终端窗口分别运行。
依赖已经装好，启动时【不要】再运行 pip install / npm install（会卡在失效代理上）。
后端端口使用 8001（已与前端 vite.config.js 对齐）。

请使用 PowerShell 终端（不是 cmd）。下面每一段【整段复制】粘贴到对应终端即可。


------------------------------------------------------------
  第零步：启动存储系统（模块三：MongoDB + Redis）
------------------------------------------------------------

本项目把清洗后的数据同步写入 MongoDB + Redis 两个存储系统。
MongoDB 和 Redis 都已配置为 Windows 服务，开机自动常驻，平时【无需手动启动】。

· MongoDB：本机已作为服务运行（端口 27017），开机自启。

· Redis：项目自带便携版已装成 Windows 服务（服务名 Redis-AIKB，端口 6379），
          开机自启、崩溃自动重启，无需开终端窗口。

平时这一步什么都不用做，直接跳到第一步即可。

【如何确认 Redis 在跑】（可选）：
    Get-Service Redis-AIKB
  状态为 Running 即正常。手动控制可用：
    Start-Service Redis-AIKB     # 启动
    Stop-Service  Redis-AIKB     # 停止
    Restart-Service Redis-AIKB   # 重启

【首次安装服务】（仅需做一次；若已装过请跳过）：
  以【管理员身份】打开 PowerShell，整段粘贴：

cd E:\aiOperation\ai_knowldge_database\tools\redis
.\redis-server.exe --service-install redis.service.conf --service-name Redis-AIKB
Set-Service -Name Redis-AIKB -StartupType Automatic
Start-Service -Name Redis-AIKB
Get-Service Redis-AIKB

  最后一行显示 Running 即安装成功，之后每次开机自动启动。

（提示：若两个存储系统暂时没起，后端仍可正常运行，只是采集时跳过入库；
  数据始终会存到 CSV，不会丢失。）


------------------------------------------------------------
  第一步：启动后端（粘贴到【后端终端】）
------------------------------------------------------------

cd E:\aiOperation\ai_knowldge_database\backend
$env:HTTP_PROXY=""; $env:HTTPS_PROXY=""
Get-NetTCPConnection -LocalPort 8001 -State Listen -ErrorAction SilentlyContinue | ForEach-Object { Stop-Process -Id $_.OwningProcess -Force -ErrorAction SilentlyContinue }
python -m uvicorn main:app --port 8001

成功标志：看到下面这行，并且【保持窗口不要关】
    INFO:     Uvicorn running on http://127.0.0.1:8001 (Press CTRL+C to quit)

（第2行：清掉失效代理，避免 pip/爬虫连不上；
  第3行：自动释放被占用的 8001 端口，避免报 10048 错误）


------------------------------------------------------------
  第二步：启动前端（粘贴到【前端终端】，新开一个窗口）
------------------------------------------------------------

cd E:\aiOperation\ai_knowldge_database\frontend
npm run dev

成功标志：看到 Local: http://localhost:5173/ ，同样【保持窗口不要关】


------------------------------------------------------------
  第三步：打开系统
------------------------------------------------------------

浏览器访问：http://localhost:5173

登录账号：
    超级用户：superadmin / super123 （可新增管理员/普通用户 + 采集 + 查看 + AI问答）
    管理员：  admin / admin123       （可新增普通用户 + 采集 + 查看 + AI问答）
    普通用户：user / user123         （仅查看分析 + AI问答，不能采集、不能管理用户）


------------------------------------------------------------
  AI问答需要本地大模型（可选）
------------------------------------------------------------

若希望 AI 回答由本地大模型生成，确保 Ollama 已在运行：
    ollama serve           （若它没有作为后台服务自启）
未启动 Ollama 时，系统会自动改用“规则统计”回答，功能仍可正常使用。


------------------------------------------------------------
  Spark 分布式分析（模块五，无需额外启动）
------------------------------------------------------------

“分析结果”页底部的“运行 Spark 分布式分析”按钮，会在后端进程内启动本地 Spark
（依赖你已安装的 JDK17，路径 E:\JDK17）。首次点击需启动 Spark 约十几秒，请耐心等待。
若 Spark 启动失败，系统会自动用 pandas 等价计算兜底，结果照常显示。


------------------------------------------------------------
  常见问题
------------------------------------------------------------

1) 报 10048 端口被占用：
   后端启动块里第3行已自动释放 8001。若仍报错，把端口换成 8000：
       python -m uvicorn main:app --port 8000
   并把 frontend\vite.config.js 里的 target 改成 http://127.0.0.1:8000

2) pip / 终端卡住反复 “Cannot connect to proxy / 10061”：
   说明走了失效代理。本说明的后端启动块第2行已清掉代理。
   切记启动时不要再敲 pip install（依赖早已装好）。

3) AI回答提示 502 / 连不上 Ollama：
   是系统代理拦截了本地请求。代码已用 trust_env=False 绕过；
   若仍异常，确认 Ollama 服务在运行（ollama list 能看到模型）。

4) 前端打不开 / 接口报错：
   确认后端窗口还开着且停在 “Uvicorn running on ...8001”。

5) “爬虫基本设置”页存储系统状态显示“未连接”：
   - MongoDB 未连接：确认 MongoDB 服务在运行（端口 27017）。
   - Redis 未连接：Redis 已是 Windows 服务，正常应自启。运行 Get-Service Redis-AIKB 看状态，
     若不是 Running 就执行 Start-Service Redis-AIKB；若提示找不到服务，按【第零步·首次安装服务】装一次。
   存储未连接不影响系统运行，数据仍会写入 CSV；连上后重新采集即可入库。


------------------------------------------------------------
  如何停止
------------------------------------------------------------

在对应终端窗口按 Ctrl + C，或直接关闭该终端窗口。


============================================================
  备注：日常只需 第一步(后端) → 第二步(前端)。
        MongoDB 和 Redis 都已是 Windows 服务、开机自启，第零步通常无需任何操作。
        （仅在第一次配置、或服务被删除时，才需按第零步重装 Redis 服务。）
